# 해야할 것
1. LH, ApplyHome DB 저장시에 가져오는 데이터 형태가 달라 테이블 분리 여부
- LH는 공고 조회라 청약 시작일, 종료일이 아닌 공고 시작, 종료일임.
- LH는 ApplyHome과 다르게 청약 시작일 정보가 없기에 현재 청약 상태가 "공고중"인 경우 알림에 포함됨

2. 따라서 구현을 어떻게 해야할지 결정 후 수정해야함 
- 아예 LH는 Web Adapter로 조회할지 (기존에 구현했던 LH 청약캘린더 내부 API 호출 방식)
- 아니면 ApplyHome도 시작일 기준이 아닌, 시작일 ~ 종료일 내에 포함되면 알림에 포함할지

### 결론 
- LH는 Web Adapter로 조회하여 구현할지 vs ApplyHome을 시작일 ~ 종료일 내에 포함되면 알림에 포함할지
- LH, ApplyHome DB 테이블 분리하거나 null 데이터가 존재하더라도 하나의 테이블로 유지할지
